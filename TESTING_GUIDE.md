# 手动测试指南

## 快速开始

### 1. 启动服务
```bash
# 启动前端
npm run dev
# 访问 http://localhost:5173

# 启动后端（可选，用于评测功能）
npm run server
```

## 2. 基础功能测试

### 简历解析
1. 打开首页，上传 PDF/DOCX 简历
2. 观察处理流程和 JSON 结果
3. 再次上传同一文件，验证缓存（应秒出）

### 批量处理
1. 切换到"批量上传"标签
2. 选择多个简历文件
3. 观察进度条

### 评测功能
1. 访问 `/evaluation` 页面
2. 创建新评测任务
3. 查看结果和指标

## 3. 验证改进效果

### 错误处理（新增）
- 关闭后端，尝试创建评测 → 应显示友好错误提示

### 场景提示词（新增）
- 上传应届生/技术/高管简历 → 观察提取结果差异

### 缓存清理（新增）
- 开发者工具 → IndexedDB → 查看 `resume-cache`

### 并发控制（新增）
- 批量上传 10 个文件 → Network 面板观察同时只有 2 个请求

## 4. 代码质量验证

```bash
npm run lint    # 0 errors, 37 warnings
npm run format  # 格式化代码
npm run build   # 7-8 秒完成
npm test        # 运行测试
```

## 5. 性能测试

- 缓存命中：第二次上传应 <100ms
- 批量处理：平均 3-10 秒/份（取决于难度）

测试文件位置：`test-resumes/` 目录
